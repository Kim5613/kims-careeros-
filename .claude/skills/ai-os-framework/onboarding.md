# 新 agent 接入路径

本文档是新 agent 或新项目接入记忆系统的最短路径，不是完整操作手册。需要细则时回查 [project-knowledge-base.md](project-knowledge-base.md)。按四步走即可：

---

## Step 1：读必读板块

读 [SKILL.md](SKILL.md) 第一节"必读"，理解六个核心概念：

核心命题、三个差距、输出质量公式、三层架构、协作闭环、积累原则。

读完后你应该明白：记忆层不是为了记更多，是为了让下次判断更快更准。

---

## Step 2：按冷启动最小集建项目文档

在 `memory/` 下创建 `project-xxx.md`，只填四个字段：

```markdown
# [项目名]

## 一句话定位
这个项目是什么、解决什么问题

## 当前状态
做到哪了、接下来做什么

## 架构概览
技术栈：[已确定的技术栈]

## 待办与优先级
1. [第一个需求]
```

同时在 `MEMORY.md` 里加一行索引：

```markdown
[项目名] 一句话描述 | 技术栈 | 当前状态
→ 项目文档: memory/project-xxx.md
```

到此为止，记忆系统已经能用了。

---

## Step 3：日常遵守两条底线

不需要执行全套协议，只要做到这两条：

**先检索，再回答。** 涉及项目状态、技术细节、历史决策时，先查项目文档和记忆，不凭推测回答。没把握就标注"待确认"。

**做完就记。** 只记四类事，其他不记：

- 做了重要决定 → 写入决策记录
- 踩坑并找到解法 → 写入技术坑点
- 模块开发完成 → 更新模块清单状态
- 收到新需求或需求变更 → 写入待办与优先级

判断标准：这条信息下次能不能让我少走弯路？能就记，不能就不记。

---

## Step 4：跑顺后开启完整协议

当项目稳定推进、积累了一定量的决策和坑点记录后，逐步升级：

1. 读 [project-knowledge-base.md](project-knowledge-base.md) 了解全部协议
2. 启用会话启动协议（检索待办、主动反馈）和会话结束协议（当场写入关键节点）
3. 创建每日巡检 cron（查漏补缺）和每周复盘 cron（提炼方法论）
4. 把反复出现的操作沉淀为 Skill

升级的判断标准：如果你开始觉得"上次踩的坑这次又踩了"，说明该升级了。

---

## 升级路径总结

```
冷启动（4 个字段 + 两条底线）
  ↓ 项目稳定推进
完整协议（启动/结束/沉淀规则/置信度标记）
  ↓ 积累足够经验
自动化（巡检 cron + 复盘 cron + Skill 沉淀）
```

不要一上来就全套。先用起来，再完善。

---

## 首次启动执行清单

读完本文档后，立即按以下顺序执行。每一步都有明确的产出物，做完打勾。

### 1. 向用户确认项目基本信息

问用户三个问题（可以用 AskUserQuestion 工具）：
- 项目名称是什么？
- 一句话描述这个项目做什么、解决什么问题？
- 当前用的什么技术栈？

拿到答案后进入下一步。

### 2. 创建项目文档

在记忆目录（通常是 `memory/` 或 agent 工作目录下的对应位置）创建 `project-{项目名}.md`，写入以下内容：

```markdown
# {项目名}

## 一句话定位
{用户提供的一句话描述}

## 当前状态
刚立项，尚未开始开发

## 架构概览
技术栈：{用户提供或推断的技术栈}

## 待办与优先级
1. {如果用户提到了第一个需求，写在这里；否则写"待用户确认"}
```

### 3. 创建或更新 MEMORY.md

如果 MEMORY.md 不存在，创建并写入：

```markdown
## 活跃项目
[{项目名}] {一句话描述} | {技术栈} | 刚立项
→ 项目文档: memory/project-{项目名}.md

## 沉淀规则
- 只记可复用的判断，不记流水账
- 项目细节落项目文档，这里只放指针
- 踩坑经验提炼为 Skill，不散落在日志里
- 会话开始前先检索上下文，不凭记忆推测
```

如果 MEMORY.md 已存在，在"活跃项目"下追加一行索引即可。

### 4. 向用户确认搭建完成

告诉用户：
> 记忆系统已搭建完成。项目文档在 memory/project-{项目名}.md，索引在 MEMORY.md。
> 接下来我们正常开始工作，我会在每次涉及项目状态时先检索文档，做完重要决策或踩坑后当场记录。

### 5. 开始工作

从此刻起，执行 Step 3 的两条底线：
- 涉及项目信息 → 先查文档再回答
- 产生决策/踩坑/模块完成/新需求 → 当场写入项目文档

不需要做任何额外操作。系统已经在跑了。
