# BookMark（示例项目文档）

> 这是一个虚构的项目文档示例，展示记忆系统运行几周后的完整状态。搭建新项目时可参照这个格式和写法。

## 一句话定位

个人书签管理工具，为内容创作者提供网页收藏、标签分类和全文检索。

## 当前状态

收藏和标签模块已完成，核心功能可用。当前卡点是全文检索的中文分词精度不够，需要调研分词库选型。

## 架构概览

技术栈：Next.js 14（App Router）+ Tailwind CSS + Drizzle ORM + SQLite

关键目录：
- `src/app/` — 页面路由（App Router）
- `src/components/` — 共享组件
- `src/lib/` — 工具函数、数据库客户端
- `src/db/schema.ts` — 数据模型

模块依赖：认证模块 → 书签模块 → 标签模块 → 搜索模块

配置要点：
- SQLite 文件存放在 `data/` 目录，需要在 `.gitignore` 中排除
- Tailwind 使用 JIT 模式，开发时热更新正常，生产构建需要确认 `content` 路径覆盖所有组件

## 决策记录

**2026-06-18 | [技术选型] 选 SQLite 而非 PostgreSQL**
个人项目数据量小（预计 <10 万条书签），SQLite 零运维、单文件部署更省心。放弃 PostgreSQL 意味着放弃了未来多用户扩展性，但当前阶段不需要。

**2026-06-15 | [架构] 网页摘要用服务端抓取，不用第三方 API**
自己用 `cheerio` 解析 meta 标签，不依赖外部 API。代价是部分网站的反爬策略会导致抓取失败，但换来了零额外成本和零数据外泄。

**2026-06-10 | [技术选型] 选 Tailwind CSS 而非 CSS Modules**
Tailwind 的 utility-first 对 AI 生成代码更友好（类名即样式，AI 不容易写出冲突的 CSS）。代价是 HTML 中类名较长，可读性略差。

## 技术坑点

**URL 抓取被目标网站 403 拒绝**
问题：部分网站（如微信公众号文章）返回 403，无法抓取标题和摘要。
根因：目标网站的反爬策略，服务端请求缺少浏览器特征的 User-Agent。
解法：请求头加上完整的浏览器 UA，并对 403 的情况降级为手动输入标题。

**SQLite WAL 模式并发写入冲突**
问题：多个请求同时写入时报 `database is locked`。
根因：SQLite 默认的 journal 模式不支持并发写入。
解法：连接时开启 WAL 模式（`PRAGMA journal_mode=WAL`），允许读写并发。

**Tailwind purge 误删动态类名**
问题：生产环境下部分样式丢失。
根因：Tailwind 的 tree-shaking 无法识别模板字符串拼接的类名（如 `bg-${color}-500`）。
解法：用 `safelist` 配置保留动态类名，或改为完整类名映射对象。
